## Project Overview

ECP Calculator is a React web app for calculating Energy Conservation Points (ECP) under NBC 2020. Builders and building officials use it to determine if a residential design meets Tier 2 (10 pts) or Tier 3 (20 pts) prescriptive energy performance thresholds for Climate Zone 6 (Nova Scotia).

## Key Files

| File | Role |
|------|------|
| `src/App.jsx` | Root component — owns all application state |
| `src/App.css` | All styles — CSS custom properties, dark/light theming |
| `src/data/ecpData.js` | API layer — imports generated JSON + compute module |
| `src/components/WallBuilder.jsx` | Wall assembly builder (simple + builder modes) |
| `src/components/WallSection.jsx` | SVG wall cross-section diagram |
| `src/components/CategoryCard.jsx` | Standard category selection card |
| `src/components/PrintSummary.jsx` | Print-only ECP summary table (Save/Print feature) |
| `scripts/generate.js` | Data pipeline: YAML → JSON lookups + audit workbook |
| `scripts/compute.js` | Thermal RSI formulas (shared between pipeline and React app) |
| `data/materials/*.yaml` | Source of truth for all material properties |
| `src/data/generated/*.json` | Pre-computed lookups (committed, regenerated by pipeline) |

## Development Commands

```bash
npm install              # Install dependencies (once)
npm run dev              # Dev server at localhost:5173
npm run build            # Production build to dist/
npm test                 # Run all tests (Vitest, 384 tests)
npm run generate         # Rebuild JSON from YAML materials
npm run generate:audit   # Generate + compare against previous JSON
```

## Architecture

- **All state in App.jsx** — no context providers, no state libraries
- **YAML-driven data pipeline** — material properties in YAML, build script generates JSON, React app imports JSON
- **CSS variables for theming** — dark mode inverts gray scale via custom properties
- **Progressive disclosure** — wall builder shows only relevant fields per wall type
- **Dual wall input modes** — direct RSI selection or assembly-based calculation
- **Client-side Excel export** — ExcelJS dynamically imported, no server component

## Data Flow

```
data/materials/*.yaml → scripts/generate.js → src/data/generated/*.json → ecpData.js → React components
```

After editing any YAML file, run `npm run generate` to regenerate JSON, then `npm test` to validate.

## Wall Types

The wall builder supports five assembly types:
- **Single stud** (wood) — parallel-path calculation + boundary layers
- **Double stud** (wood) — two stud rows + isothermal gap
- **Service wall** — interior 2x4 service cavity (toggle on single/double stud)
- **Steel frame** — NBC K-factor weighted method
- **ICF** — insulated concrete forms, pure series sum

## ECP Categories

Eight scoring categories, each with point thresholds from NBC 2020 Tables 9.36.8.4–9.36.8.10:
- Above-Ground Walls, Below Grade Walls, Windows/Doors, Airtightness
- Hot Water (Electric), Hot Water (Non-Electric) — mutually exclusive
- Ventilation (HRV/ERV), Heated Volume

## Conventions

- When modifying YAML materials, always regenerate JSON and commit both
- When modifying thresholds in `generate.js`, regenerate and run tests
- CSS changes should use existing custom properties where possible
- Wall builder uses FieldGroup numbered cards for visual grouping
- ExcelJS is a dev dependency but dynamically imported at runtime (code-split)
